# 轻量级量化本地数据库设计方案 (QuantDatabase)

> **版本**：v0.1.0  
> **设计目标**：面向个人量化开发者，构建一个跨市场、多品种、可扩展、易于维护的本地金融数据库。  
> **核心理念**：轻量起步，标准先行，元数据自描述，代码即文档。

---

## 1. 项目愿景

建立一个本地化的量化研究数据库（QuantDatabase），支持 **A股、港股、美股** 的**股票、ETF、指数、期货、期权** 等主要品种的日常行情与基础数据。数据库应具备以下特性：

- **轻量跨平台**：可在 Linux / macOS / Windows 上零摩擦部署与迁移。
- **纯 SQL 访问**：支持命令行、Python、其他语言通过标准接口进行 SQL 查询与分析。
- **多数据源整合**：能同时接入 Tushare、yfinance、AKShare 等不同数据渠道，并统一管理。
- **元数据自描述**：数据库的结构、来源、更新历史等全部以表的形式存储在库内，做到“看到库就知道一切”。
- **代码规范化**：使用 Python `dataclass` 定义所有数据表的 Schema，实现建表、数据拉取、字段映射的一致性。

---

## 2. 技术选型

### 2.1 核心数据库：DuckDB

**选型理由**  

- **零配置、文件级便携**：一个 `.db` 文件即完整数据库，复制、备份、迁移极其方便，完美契合“本地轻量”的要求。
- **原生支持直接查询 Parquet/CSV 文件**：未来历史数据可归档成 Parquet 格式，DuckDB 能直接在上面跑 SQL，分析成本极低。
- **极致单机分析性能**：面向列式存储、向量化执行引擎，在聚合、窗口等金融分析场景下显著快于 SQLite / MySQL 等。
- **完整 SQL 支持**：支持关联、子查询、窗口函数、CTE、自建函数等，完全满足量化研究的查询需求。
- **跨平台**：官方预编译二进制覆盖 x86/ARM 的 Linux、macOS、Windows。

**使用方式**  

- 命令行交互：`duckdb quant.db`
- Python 集成：`import duckdb; con = duckdb.connect('quant.db')`
- 未来可扩展：通过少量 Python 代码包装成 REST API，供远程调用。

### 2.2 辅助组件（按需引入）


| 组件                           | 用途            | 引入阶段              |
| ---------------------------- | ------------- | ----------------- |
| **Parquet 文件**               | 历史数据冷归档、跨项目共享 | 数据量增大后            |
| **Redis**                    | 实时行情缓存、消息队列   | 构建实盘/准实时系统时       |
| **TimescaleDB / ClickHouse** | 超大规模历史数据的深层分析 | 单机 DuckDB 遇到性能瓶颈时 |


在当前阶段，**一个 DuckDB 即为全部**。

## 3. 数据库模式设计

### 3.1 通用命名约定

- 数据表名统一采用小写蛇形命名法：`stock_daily`, `etf_adj_nav`。
- 所有市场、品种通用唯一标识字段：`instrument_id` （建议格式：`"000001.SZ"`, `"AAPL"`, `"HSI"`）。
- 金额单位：统一为**元**，如果原始数据为千元，入库时乘以 1000。
- 日期字段存储为 `datetime.date` 类型。

### 3.2 核心数据表

初期从 **A 股日线行情** 与 **股票基本信息** 开始，分批次扩展。

#### 3.2.1 股票日线行情 `stock_daily`


| 字段              | 类型      | 说明     | 示例            |
| --------------- | ------- | ------ | ------------- |
| `instrument_id` | VARCHAR | 股票代码   | `'000001.SZ'` |
| `trade_date`    | DATE    | 交易日期   | `2024-01-02`  |
| `open`          | DOUBLE  | 开盘价（元） | 10.50         |
| `high`          | DOUBLE  | 最高价    | 10.80         |
| `low`           | DOUBLE  | 最低价    | 10.20         |
| `close`         | DOUBLE  | 收盘价    | 10.55         |
| `pre_close`     | DOUBLE  | 前收盘价   | 10.00         |
| `change`        | DOUBLE  | 涨跌额    | 0.55          |
| `pct_chg`       | DOUBLE  | 涨跌幅（%） | 5.5           |
| `vol`           | DOUBLE  | 成交量（手） | 100000        |
| `amount`        | DOUBLE  | 成交额（元） | 1050000       |


#### 3.2.2 股票基本信息 `stock_info`


| 字段              | 类型      | 说明                |
| --------------- | ------- | ----------------- |
| `instrument_id` | VARCHAR | 股票代码              |
| `name`          | VARCHAR | 股票名称              |
| `market`        | VARCHAR | 市场（'A','HK','US'） |
| `list_date`     | DATE    | 上市日期              |
| `delist_date`   | DATE    | 退市日期（可为空）         |


#### 3.2.3 ETF、指数、期货、期权等

后续按统一模式扩展，如 `etf_daily`、`index_daily`、`futures_daily`、`option_daily` 等，字段结构与行情类表保持一致。

### 3.3 元数据管理表（系统自描述）

**这是本项目的“灵魂设计”**，让数据库具备自我解释能力。

#### 3.3.1 数据表目录 `catalog_tables`


| 字段                 | 类型                  | 说明                            |
| ------------------ | ------------------- | ----------------------------- |
| `table_name`       | VARCHAR PRIMARY KEY | 表名，如 `stock_daily`            |
| `description`      | VARCHAR             | 表的中文描述                        |
| `source_type`      | VARCHAR             | 数据源类型：'tushare'、'yfinance'    |
| `source_api`       | VARCHAR             | 具体接口或参数：'pro.daily()'         |
| `generator_script` | VARCHAR             | 生成/更新该表的脚本路径                  |
| `update_freq`      | VARCHAR             | 更新频率：'daily'、'weekly'         |
| `last_updated`     | TIMESTAMP           | 最后成功更新时间                      |
| `status`           | VARCHAR             | 'active'、'error'、'deprecated' |
| `notes`            | VARCHAR             | 补充说明（如字段清单）                   |


#### 3.3.2 更新日志表 `update_log`（可选）

如果需要追踪每次更新的详细记录可启用，初期非必须。

## 4. 数据管道与代码规范

### 4.1 使用 `dataclass` 作为字段唯一定义源

为每张数据表定义一个 Python `dataclass`，该 `dataclass` 统一驱动：

- **数据库建表** (`CREATE TABLE`)  
- **Tushare / yfinance 接口字段选择** (`fields` 参数)  
- **数据清洗与类型转换**

**示例：stock_daily.py**

```python
from dataclasses import dataclass, fields
from datetime import date
from typing import List

@dataclass
class StockDaily:
    instrument_id: str
    trade_date: date
    open: float
    high: float
    low: float
    close: float
    pre_close: float
    change: float
    pct_chg: float
    vol: float
    amount: float
```
